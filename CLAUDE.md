## Project Overview

RSS feed aggregator that fetches feeds from various sources (sports, news, tech, etc.), writes them to `data/feeds.json`, and displays them via a static `index.html` hosted on GitHub Pages.

## Commands

- **Install deps:** `npm install`
- **Fetch feeds:** `node fetch-feeds.js` (writes to `data/feeds.json`)

No build step, test suite, or linter.

## Architecture

- `fetch-feeds.js` — Node.js script that fetches all RSS feeds in parallel using `rss-parser`, with per-feed timeouts (6s hard limit, 5s parser timeout). Feed sources are defined inline as the `feeds` object (keyed by category). Output is `data/feeds.json`.
- `index.html` — Single-file frontend (HTML/CSS/JS, no framework). Loads `data/feeds.json` at runtime and renders feeds with category filter buttons. Dark theme with team-colored accents.
- `.github/workflows/deploy.yml` — Runs every 8 hours (and on push to main): fetches feeds, then deploys `index.html` + `data/` to GitHub Pages.

## Key Details

- Adding a new feed category requires changes in three places: the `feeds` object in `fetch-feeds.js`, the filter buttons in `index.html`, and the CSS color variables/rules in `index.html`.
- Feed items are capped at 15 per source (fetch) and 5 per source (display).
- `data/feeds.json` is gitignored in practice (generated by CI) but may exist locally after running `fetch-feeds.js`.
